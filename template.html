How constitutional AI systems can dynamically adapt instructional experiences to diverse cognitive profiles while maintaining safety, usability, trustworthiness, measurable learning outcomes, and engineering robustness — with neurodivergent individuals as co-creators, decision-makers, and the primary evaluators of success — bootstrappable from a solo prototype.

Executive Summary (NSF Project Summary Format)

Intellectual Merit

This template co-designs, builds, and validates an AI-driven research ecosystem that is constitutionally safe, cognitively adaptive, neurodivergent-led, and bootstrappable from a solo researcher's laptop. The system dynamically adapts instruction to diverse cognitive profiles while maintaining safety, trustworthiness, and production-grade engineering robustness, with neurodivergent individuals as co-creators, decision-makers, and primary evaluators across four culturally distinct global hubs.

Key innovations: a continuously audited Constitutional Compliance Index (CCI), alignment faking safeguards with live stress-testing, and a planned open-source prototype that fully instantiates every core theoretical construct. A dedicated scaling architecture analysis identifies five critical bottlenecks and enumerates concrete, phased mitigations.

Broader Impacts

The template embeds neurodivergent-first governance as a model for inclusive AI research. An open-source starter kit, reproducible artifact package, and public dashboards advance public scientific literacy. The phased funding strategy (bootstrapped <$1,000 → intermediate $120K → mid-scale $500K → megaproject $6.8M) demonstrates resource-efficient innovation transferable to low-resource settings and creates a deterministic, laddered pathway for any researcher or small enterprise to produce Tier-1 level work.

Neurodivergent-First Core Principles

  1. Choice & Control — Participants can opt in/out of any data collection, customize their interface, and influence governance rules
  2. Sensory Accommodation — High-contrast CSS, ARIA, three interaction modes (text, voice, visual icons)
  3. Communication Flexibility — All materials in plain language, visual, audio. Optional text-to-speech via Web Speech API
  4. Executive Function Support — Visual schedules, task breakdowns, universal "pause" button
  5. Privacy & Transparency — Local-first data architecture. No raw data leaves the browser without explicit opt-in
  6. Community Ownership — NAB, CGB, CHAE each have defined scopes of authority with published precedence order
  7. Iterative Feedback — "How was this?" button linked to a free database at every tier
  8. Self-Evolution — Constitutional rules and adaptation profiles evolve based on community-driven proposals and open-source PRs

Pre-Registered Hypotheses

  • H1 (Adaptation Benefit) — NAP-driven adaptation reduces intra-task cognitive load variance compared to static baseline
  • H2 (Trust Calibration) — TCE smaller for adaptive condition; moderated by NAP profile ownership
  • H3 (Safety Drift Suppression) — Continuous CHAE + adversarial testing maintains CCI >0.95; periodic review drifts below 0.90 over 12 weeks
  • H4 (Cross-Cultural Invariance) — Autonomy Index and Sensory Comfort Score demonstrate scalar invariance across four hubs
  • H5 (Neurodivergent Empowerment) — Sites with Neurodivergent Co-Lead + NAB veto show higher community influence and lower dropout
  • H6 (Bootstrapping Feasibility) — Bootstrapped pilot demonstrates acceptable feasibility metrics; effect sizes treated as preliminary
  • H7 (Community Self-Evolution) — OSSC-merged enhancements positively correlate with CCI stability over 18 months
  • H8 (CHAE Incremental Safety) — CHAE real-time logs contain substantial proportion of safety events not flagged by automated red-teaming
